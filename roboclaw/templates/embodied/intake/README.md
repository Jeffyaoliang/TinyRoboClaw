# Intake Notes

Create one note per setup under this directory before generating code.

For a first-time run, this note is the source of truth RoboClaw should use
before writing any assembly, deployment, adapter, or simulator files.

Recommended sections:

- Robot model and embodiment type
- Sensors and mounting points
- ROS2 packages, namespaces, topics, services, and actions
- Real targets and simulator targets
- Connection details such as serial devices, IPs, camera ids, and calibration paths
- Safety limits, reset/home behavior, and known debug steps
- What should count as a successful first run for this setup
